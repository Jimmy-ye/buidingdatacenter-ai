## 1. 系统概述

### 1.1 功能介绍

BDC-AI PC-UI 是基于 NiceGUI 开发的桌面端管理界面，提供以下功能：

- ✅ **项目管理**：创建、查看、编辑、删除项目
- ✅ **工程结构管理**：管理建筑、系统、设备树形结构
- ✅ **资产管理**：上传、查看、管理多模态资产（图片、表格等）
- ✅ **AI 分析**：调用 GLM-4V 进行图片识别和场景分析
- ✅ **数据导出**：导出项目数据和分析报告
- ✅ **用户管理**：管理用户、角色和权限（管理员）

### 1.2 界面布局

PC-UI 采用现代化的 Web 风格界面：

```
┌─────────────────────────────────────────────────────┐
│  🏠 BDC-AI        搜索框         👤 用户信息 ⚙️    │ ← 顶部导航栏
├──────────┬──────────────────────────────────────────┤
│          │                                           │
│  📁 项目  │           主工作区                      │
│  📁 建筑  │                                           │
│  📁 系统  │    内容展示、表单、数据表格...           │
│  📁 设备  │                                           │
│  📁 资产  │                                           │
│  📁 设置  │                                           │
│          │                                           │
└──────────┴──────────────────────────────────────────┘
  侧边栏           主内容区
```

### 1.3 系统要求

**硬件要求**：
- CPU：双核及以上
- 内存：4 GB 及以上
- 硬盘：500 MB 可用空间
- 显示器：分辨率 1280×720 及以上

**软件要求**：
- 操作系统：Windows 10/11、macOS 10.15+、Ubuntu 20.04+
- Python：3.11 及以上
- 浏览器：Chrome 90+、Edge 90+、Safari 14+、Firefox 88+（推荐使用 Chrome）

## 2. 安装和启动

### 2.1 环境准备

#### 安装 Python

**Windows**：

1. 访问 https://www.python.org/downloads/
2. 下载 Python 3.11 Windows installer
3. 运行安装程序，**务必勾选 "Add Python to PATH"**
4. 验证安装：打开命令提示符，输入 `python --version`

**macOS**：

```bash
# 使用 Homebrew 安装
brew install python@3.11

# 验证安装
python3.11 --version
```

**Ubuntu**：

```bash
# 添加 Python 3.11 PPA
sudo add-apt-repository ppa:deadsnakes/ppa
sudo apt update

# 安装 Python 3.11
sudo apt install python3.11 python3.11-venv

# 验证安装
python3.11 --version
```

### 2.2 安装依赖

#### 克隆项目（如未克隆）

```bash
git clone https://github.com/Jimmy-ye/buidingdatacenter-ai.git
cd buidingdatacenter-ai
```

#### 安装 Python 依赖

```bash
# 创建虚拟环境（推荐）
python -m venv venv

# 激活虚拟环境
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate

# 安装依赖
pip install -r services/backend/requirements.txt
```

### 2.3 配置后端地址

PC-UI 需要连接到后端 API。

**开发环境**：

```bash
# 默认连接到本地后端
BDC_BACKEND_BASE_URL=http://localhost:8000
```

**生产环境**：

编辑 `desktop/nicegui_app/.env`：

```bash
BDC_BACKEND_BASE_URL=http://your-server:8000
```

### 2.4 启动 PC-UI

#### 开发模式

```bash
# 确保虚拟环境已激活
cd desktop/nicegui_app
python pc_app.py
```

启动成功后，浏览器会自动打开：http://localhost:8080

#### 生产模式（Linux systemd）

创建 `/etc/systemd/system/bdc-pcui.service`：

```ini
[Unit]
Description=BDC-AI PC-UI Service
After=network.target bdc-backend.service

[Service]
Type=simple
User=bdc_ai
WorkingDirectory=/opt/bdc-ai/desktop/nicegui_app
Environment="PATH=/opt/bdc-ai/venv/bin"
ExecStart=/opt/bdc-ai/venv/bin/python pc_app.py
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl enable bdc-pcui
sudo systemctl start bdc-pcui
```

## 3. 登录和认证

### 3.1 用户登录

#### 默认管理员账号

```
用户名：admin
密码：admin123
角色：超级管理员
```

⚠️ **重要**：首次登录后请立即修改默认密码！

#### 登录步骤

1. 打开浏览器访问：http://localhost:8080
2. 进入登录页面
3. 输入用户名和密码
4. 点击 "登录" 按钮
5. 登录成功后自动跳转到主页面

#### 记住密码

勾选 "记住密码" 复选框，下次访问时自动登录（Token 有效期 7 天）。

### 3.2 权限管理

#### 用户类型

| 角色 | 权限级别 | 可访问功能 |
|-----|---------|-----------|
| 超级管理员 | 100 | 所有功能，包括用户管理 |
| 管理员 | 80 | 项目管理、资产管理、用户管理 |
| 项目经理 | 60 | 项目 CRUD、资产查看/上传、报告生成 |
| 数据分析师 | 50 | 数据查看、报告生成/导出 |
| 现场工程师 | 40 | 资产上传、项目查看 |
| 访客 | 20 | 只读权限 |

#### 功能权限

**项目管理**：
- 查看：所有角色
- 创建：管理员及以上
- 编辑：项目经理及以上
- 删除：管理员及以上

**资产管理**：
- 查看：所有角色
- 上传：工程师及以上
- 删除：管理员及以上

**用户管理**：
- 仅管理员和超级管理员可访问

## 5. 核心功能

### 5.1 项目管理

#### 项目列表

**访问路径**：侧边栏 → 📇 项目管理

**显示内容**：

| 列名 | 说明 |
|-----|------|
| 项目名称 | 项目的名称 |
| 客户 | 客户名称 |
| 位置 | 项目所在地 |
| 类型 | commercial/industrial/public_building |
| 状态 | planning/in_progress/completed |
| 创建时间 | 项目创建日期 |
| 操作 | 编辑、删除按钮 |

**筛选和搜索**：

- 按状态筛选：全部/规划中/进行中/已完成/暂停
- 按类型筛选：全部/商业/工业/公共建筑/数据中心
- 搜索：按项目名称或客户名称模糊搜索

**排序**：点击列标题进行升序/降序排序

#### 创建项目

**步骤**：

1. 点击项目列表右上角 "➕ 新建项目" 按钮
2. 填写项目信息：
   - 项目名称（必填）
   - 客户名称（必填）
   - 项目位置
   - 项目类型（下拉选择）
   - 项目状态（下拉选择）
   - 项目描述
3. 点击 "保存" 按钮

**验证**：
- 项目名称和客户名称不能为空
- 项目类型和状态必须从下拉列表选择
- 保存成功后自动跳转到项目详情页

#### 编辑项目

**步骤**：

1. 在项目列表中找到要编辑的项目
2. 点击操作列的 "✏️ 编辑" 按钮
3. 修改项目信息
4. 点击 "保存" 按钮

**说明**：只有项目经理及以上角色可以编辑项目。

#### 删除项目

**步骤**：

1. 在项目列表中找到要删除的项目
2. 点击操作列的 "🗑️ 删除" 按钮
3. 在确认对话框中输入删除原因
4. 点击 "确认删除"

**说明**：
- 删除是软删除，数据不会真正删除
- 只有管理员及以上角色可以删除项目
- 删除的项目可以在回收站恢复

### 5.2 工程结构管理

#### 查看工程结构树

**访问路径**：侧边栏 → 🏢 工程结构

**显示内容**：

```
📦 项目根
  └─ 🏢 A座办公楼
       ├─ ⚙️ HVAC系统
       │    ├─ 🔧 冷机CH-001
       │    ├─ 🔧 冷机CH-002
       │    └─ 🔧 风机盘管FCU-03
       ├─ ⚙️ 照明系统
       └─ 🏢 5F办公区 (Zone)
```

**操作**：

- 点击节点展开/折叠子节点
- 点击节点查看详情
- 右键点击节点显示上下文菜单：
  - 新建子节点
  - 编辑节点
  - 删除节点
  - 查看关联资产

#### 创建建筑

**步骤**：

1. 选择项目根节点
2. 右键点击 → "新建建筑"
3. 填写建筑信息：
   - 建筑名称（必填）
   - 建筑用途（office/commercial/datacenter/mixed_use）
   - 建筑面积（㎡）
   - 建成年份
   - 能效等级（1-5星）
4. 点击 "保存"

**自动创建系统**：

创建建筑后，系统会自动创建 9 个默认系统：
- 围护结构
- 制冷
- 制热
- 空调末端
- 照明
- 电梯
- 动力
- 电力监控
- 能管平台

#### 创建系统

**步骤**：

1. 选择建筑节点
2. 右键点击 → "新建系统"
3. 填写系统信息：
   - 系统类型（必填）
   - 系统名称
   - 系统描述
4. 点击 "保存"

#### 创建设备

**步骤**：

1. 选择系统节点
2. 右键点击 → "新建设备"
3. 填写设备信息：
   - 设备类型
   - 设备型号
   - 额定功率（kW）
   - 序列号
   - 所在区域（可选）
4. 点击 "保存"

**说明**：
- 设备必须归属于某个系统
- 设备可以位于某个区域（可选）
- 设备会自动关联到建筑

### 5.3 资产管理

#### 资产列表

**访问路径**：侧边栏 → 📁 资产管理

**筛选条件**：

- 项目（必选）
- 建筑系统（可选）
- 设备（可选）
- 模态类型：image/table/text/audio/document
- 时间范围：创建时间

**显示内容**：

- 缩略图（图片）
- 资产标题
- 资产描述
- 工程路径
- 创建时间
- 操作按钮

#### 上传图片资产

**步骤**：

1. 点击资产列表右上角 "➕ 上传资产" 按钮
2. 选择资产类型：图片
3. 选择文件：
   - 点击 "选择文件" 按钮
   - 或拖拽文件到上传区域
4. 填写资产信息：
   - 所属项目（必选）
   - 所属设备（可选）
   - 内容角色：scene_issue/nameplate/general
   - 文字备注
5. 点击 "上传"

**支持格式**：JPG、PNG、HEIC（最大 50 MB）

**上传后处理**：

1. 自动调用 OCR 识别图片文字
2. 自动调用 GLM-4V 进行场景分析（如选择了 scene_issue）
3. 提取的结构化信息存储到数据库

#### 查看资产详情

**步骤**：

1. 在资产列表中点击资产缩略图或标题
2. 打开资产详情页

**显示内容**：

- 原始图片
- OCR 识别的文字
- GLM-4V 分析结果（如适用）
- 工程结构信息（项目、建筑、系统、设备）
- 资产元数据（创建时间、大小、格式）

**操作**：

- 下载原始图片
- 重新运行 OCR
- 重新运行 GLM-4V 分析
- 编辑资产信息
- 删除资产

#### 批量上传

**步骤**：

1. 选择多个文件（Ctrl+点击或 Shift+点击）
2. 填写共同的资产信息（项目、设备等）
3. 点击 "批量上传"

**说明**：批量上传会逐个处理，可能需要较长时间。

### 5.4 AI 分析

#### OCR 文字识别

**功能**：从图片中提取文字信息

**使用场景**：
- 设备铭牌识别
- 仪表读数识别
- 文档扫描

**步骤**：

1. 上传图片资产
2. 系统自动调用 OCR
3. 查看 OCR 结果

**手动触发**：

1. 打开资产详情
2. 点击 "重新运行 OCR" 按钮
3. 等待处理完成
4. 查看结果

#### GLM-4V 场景分析

**功能**：使用大语言模型分析图片内容

**使用场景**：
- 现场问题识别（设备故障、异常情况）
- 能效问题分析（保温缺失、冷桥等）
- 安全隐患识别

**分析类型**：

1. **现场问题**：
   - 识别设备问题
   - 识别安装问题
   - 识别维护问题

2. **设备铭牌**：
   - 提取设备参数
   - 识别设备型号
   - 提取制造商信息

**步骤**：

1. 上传图片时选择内容角色为 "现场问题"
2. 系统自动调用 GLM-4V
3. 查看分析结果

**手动触发**：

1. 打开资产详情
2. 点击 "重新运行 AI 分析" 按钮
3. 等待处理完成（可能需要 10-30 秒）
4. 查看分析结果

**分析结果格式**：

```json
{
  "summary": "现场问题描述",
  "issues": [
    {
      "type": "设备故障",
      "description": "风机盘管异响",
      "severity": "高",
      "suggestion": "建议立即检修"
    }
  ],
  "detected_objects": ["风机盘管", "管道"],
  "confidence": 0.92
}
```

## 6. 高级功能

### 6.1 数据筛选

#### 高级筛选器

**位置**：各列表页面顶部

**功能**：

- **多条件组合**：可以同时使用多个筛选条件
- **保存筛选**：保存常用的筛选条件
- **快速切换**：在保存的筛选之间快速切换

**示例**：

筛选 "某科技" 客户的 "进行中" 项目：
1. 客户筛选：输入 "某科技"
2. 状态筛选：选择 "进行中"
3. 点击 "应用筛选"

#### 保存筛选

**步骤**：

1. 设置筛选条件
2. 点击筛选器旁的 "💾 保存" 按钮
3. 输入筛选名称
4. 点击 "保存"

**使用保存的筛选**：

1. 点击筛选器旁的 "📋 我的筛选" 按钮
2. 选择要使用的筛选
3. 筛选条件自动应用

### 6.2 数据导出

#### 导出项目数据

**支持格式**：Excel、CSV、JSON

**步骤**：

1. 在项目列表中设置筛选条件（可选）
2. 点击右上角 "📤 导出" 按钮
3. 选择导出格式
4. 选择导出字段
5. 点击 "确认导出"
6. 下载文件

**导出字段**（可多选）：

- 项目名称
- 客户
- 位置
- 类型
- 状态
- 建筑数量
- 设备数量
- 资产数量
- 创建时间
- 更新时间

#### 导出资产列表

**步骤**：

1. 在资产列表中设置筛选条件
2. 点击 "📤 导出" 按钮
3. 选择导出格式
4. 选择是否包含图片附件
5. 点击 "确认导出"

**说明**：
- 不包含图片：仅导出元数据（速度快）
- 包含图片：打包所有图片（速度慢，文件大）

### 6.3 系统设置

#### 个人设置

**访问路径**：顶部导航栏 → 👤 用户 → 个人设置

**可设置项**：

- **基本信息**：

## 8. 故障排查

### 8.1 启动问题

#### 问题：无法启动 PC-UI

**症状**：运行 `python pc_app.py` 后报错

**可能原因**：
1. Python 版本不符合要求
2. 依赖未安装
3. 端口被占用

**解决方案**：

```bash
# 检查 Python 版本
python --version
# 应该是 3.11 或更高

# 安装依赖
pip install nicegui

# 检查端口占用（默认 8080）
# Windows
netstat -ano | findstr :8080
# macOS/Linux
lsof -i :8080

# 更换端口启动
python pc_app.py --port 8081
```

#### 问题：后端连接失败

**症状**：登录页面显示 "无法连接到后端服务"

**解决方案**：

1. 检查后端服务是否运行：
```bash
curl http://localhost:8000/api/v1/health
```

2. 检查配置文件：
```bash
cat desktop/nicegui_app/.env
# 确认 BDC_BACKEND_BASE_URL 正确
```

3. 检查网络连接：
```bash
ping localhost
telnet localhost 8000
```

### 8.2 功能问题

#### 问题：上传图片失败

**症状**：上传进度条卡住或报错

**可能原因**：
1. 文件过大（超过 50 MB）
2. 网络连接中断
3. 后端存储空间不足

**解决方案**：

1. 压缩图片后重试
2. 检查网络连接
3. 清理浏览器缓存
4. 检查后端存储空间

#### 问题：AI 分析无响应

**症状**：点击 "运行 AI 分析" 后长时间无响应

**可能原因**：
1. GLM API KEY 未配置或失效
2. 网络无法访问 GLM API
3. API 配额已用尽

**解决方案**：

1. 检查 API KEY 配置：
```bash
cat .env | grep GLM_API_KEY
```

2. 测试 API 连接：
```bash
python scripts/test_glm_api.py
```

3. 查看后端日志：
```bash
sudo journalctl -u bdc-backend -f
```

### 8.3 性能问题

#### 问题：页面加载缓慢

**症状**：切换页面或加载数据时响应慢

**可能原因**：
1. 数据量过大
2. 网络延迟高
3. 后端性能问题

**解决方案**：

1. 使用筛选条件减少数据量
2. 检查网络连接
3. 清理浏览器缓存
4. 查看后端性能（可能需要数据库优化）

#### 问题：内存占用高

**症状**：浏览器标签页占用大量内存

**解决方案**：

1. 关闭不需要的浏览器标签页
2. 定期刷新页面
3. 清理浏览器缓存
4. 使用性能更好的浏览器（推荐 Chrome）

### 9.1 数据管理

#### 定期备份数据

**建议频率**：

- 一般项目：每周备份

```bash
python scripts/backup_database.sh
```

#### 清理无用数据

**建议**：
- 定期删除测试项目
- 清理重复上传的资产
- 归档已完成的项目

### 9.2 使用建议

#### 工程录入最佳实践

1. **创建建筑的顺序**：
   - 先创建建筑
   - 让系统自动创建默认系统
   - 再根据需要补充自定义系统

2. **设备命名规范**：
   - 使用统一的前缀：CH（冷机）、FCU（风机盘管）
   - 包含位置信息：FCU-5F-03（5楼第3台）
   - 包含序列号：CH-001

3. **资产上传建议**：
   - 上传前压缩图片（减少上传时间）
   - 使用有意义的内容角色
   - 添加详细的备注信息

#### AI 分析使用建议

1. **OCR 使用场景**：
   - 设备铭牌
   - 仪表读数
   - 技术文档

2. **GLM-4V 使用场景**：
   - 现场问题识别
   - 设备故障诊断
   - 能效问题分析

3. **提高分析准确度**：
   - 确保图片清晰度
   - 避免过度曝光或暗光
   - 拍摄整体场景而非局部

### 9.3 安全建议

1. **定期修改密码**：建议每 90 天修改一次
2. **使用强密码**：至少 8 位，包含大小写字母、数字、特殊字符
3. **及时登出**：使用完毕后及时退出登录
4. **保护 Token**：不要在公共场所保存登录状态
5. **权限最小化**：为用户分配最小必要权限

## 10. FAQ

### 10.1 常见问题

**Q1: 忘记密码怎么办？**

A: 联系管理员重置密码，或使用默认 admin/admin123 登录后修改。

**Q2: 可以上传视频文件吗？**

A: 当前版本仅支持图片、表格、文本文件。视频支持将在后续版本添加。

**Q3: 如何批量导入设备？**

A: 当前版本需要手动创建。批量导入功能（Excel/CSV）将在后续版本添加。

**Q4: 资产可以关联到多个设备吗？**

A: 不可以。每个资产只能关联到一个设备。如需关联多个，请分别上传。

**Q5: AI 分析的准确度如何？**

A: OCR 准确度约 95%，GLM-4V 场景分析准确度约 85-90%。建议人工复核重要信息。

**Q6: 可以离线使用吗？**

A: 不可以。PC-UI 需要连接到后端服务，所有数据存储在服务器上。

**Q7: 如何导出所有项目数据？**

A: 在项目管理页面使用 "导出" 功能，可以选择导出格式和字段。

**Q8: 资产上传有大小限制吗？**

A: 单个文件最大 50 MB。建议压缩图片后上传。

**Q9: 可以自定义字段吗？**

A: 当前版本不支持。自定义字段功能将在后续版本添加。

**Q10: 如何查看操作日志？**

A: 当前版本需要联系管理员查看数据库。操作日志界面将在后续版本添加。
